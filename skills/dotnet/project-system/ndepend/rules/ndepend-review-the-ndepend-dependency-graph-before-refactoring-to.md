## Review the NDepend dependency graph before refactoring to identify the actual dependency fan-out

Review the NDepend dependency graph before refactoring to identify the actual dependency fan-out: rather than guessing which types will be affected; the visual graph reveals transitive dependencies that text-based analysis misses.
