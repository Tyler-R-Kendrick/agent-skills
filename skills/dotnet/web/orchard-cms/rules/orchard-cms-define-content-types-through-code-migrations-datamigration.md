## Define content types through code migrations (`DataMigration`) rather than through the admin UI

Define content types through code migrations (`DataMigration`) rather than through the admin UI: so that content type definitions are version-controlled, reproducible across environments, and deployable through CI/CD pipelines without manual admin steps.
